## How it works

The Chassis - CSS accordion uses `details` element and let browser do the expanding and collapsing of content. It uses javascript to add a smooth transion. Because of `details` elements native purpose, accordion doesn't require any aria or accessibility definitions.

<Callout name="info-prefersreducedmotion" />

## Example

Click the accordions below to expand/collapse the accordion content.

Just add the `open` attribute on the `details` element to render it is expanded by default.

<Example code={`<div class="accordion">
  <details open>
    <summary>
      <h4 class="accordion-title" role="heading" aria-level="4">Accordion Item #1</h4>
    </summary>
    <div class="accordion-body">
      <p>
        This is the <strong>first item's content</strong>. It is shown by default, until another detail element gets "<b>[open]</b>" by clicked on its summary.
      </p>
    </div>
  </details>
  <details>
    <summary>
      <h4 class="accordion-title" role="heading" aria-level="4">Accordion Item #2</h4>
    </summary>
    <div class="accordion-body">
      <p>
        This is the <strong>second item's content</strong>. It is hidden by default, until clicked on its summary element.
      </p>
    </div>
  </details>
  <details>
    <summary>
      <h4 class="accordion-title" role="heading" aria-level="4">Accordion Item #3</h4>
    </summary>
    <div class="accordion-body">
      <p>
        This is the <strong>third item's content</strong>. It is hidden by default, until clicked on its summary element.
      </p>
    </div>
  </details>
</div>`} />

<Callout type="warning">
  When using heading tags (e.g., `<h4>`) inside the `summary` element, it is recommended to include the `role="heading"` attribute and an `aria-level` attribute. This ensures that screen readers correctly interpret the heading structure within the accordion. The `aria-level` should match the appropriate heading level in the document hierarchy.
</Callout>

## Accessibility

No need to aria and role definitions. Since the accordion component uses the HTML5 `details` element, accessibility features are automatically handled by the browser.

### Note on Heading Tags in Summary

When using heading tags (e.g., `<h4>`) inside the `summary` element, it is recommended to include the `role="heading"` attribute and an `aria-level` attribute. This ensures that screen readers correctly interpret the heading structure within the accordion. The `aria-level` should match the appropriate heading level in the document hierarchy.

For example:
```html
<summary>
  <h4 class="accordion-title" role="heading" aria-level="4">Accordion Item #1</h4>
</summary>
```

This approach improves accessibility by providing a clear and consistent heading structure for assistive technologies.
