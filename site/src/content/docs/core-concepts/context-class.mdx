## How it works?

Chassis - CSS design system does not write properties directly for every classes. Instead of that, it uses <a href="https://www.w3.org/TR/css-variables-1/#example-c3e4e8d9" target="_blank">repeating custom css properties</a> and extends placeholder classes to share same property definitions but values.

### Placeholder Selector

The `%context` placeholder selector uses [context mixin](#context-mixin) in a loop to create palette classes for each context color. These classes does not assign any property, only changes the context palette color variables

### Component Class

The `.context` class extends above placeholder selectors to create classes for each context color & style combinations.

<ScssDocs name="context-class" file="scss/_context.scss" />

And now we've have all context & style variants of our components.

<Example code={`<div class="context primary p-xsmall m-small">This is a primary basic context block</div>
<div class="context primary solid p-xsmall m-small">This is a primary solid context block</div>
<div class="context primary smooth p-xsmall m-small">This is a primary smooth context block</div>
<div class="context primary outline border p-xsmall m-small">This is a primary outline context block</div>`} />

## Custom Components

### Example 1

Lets say an `.example` component use a custom highlight color and falls back to context colors in failures.

<ScssDocs name="example-1" file="site/src/scss/_context.scss" />

By doing this, the color can be changed easily if we want to give more emphasis somewhere and also it fallbacks to `--cx-fg-highlight` and adapts to context in any failure.

<Example code={`<div>
  <p class="example p-xsmall">--example-fg</p>
</div>
<div style="--example-fg: var(--cx-fg-main);">
  <p class="example p-xsmall">--cx-fg-main</p>
</div>
<div style="--example-fg: var(--no-color);">
  <p class="example p-xsmall">--cx-fg-highlight</p>
</div>
<div class="context primary">
  <p class="example p-xsmall">--example-fg</p>
</div>
<div class="context primary" style="--example-fg: var(--no-color);">
  <p class="example p-xsmall">--cx-fg-highlight</p>
</div>
<div class="context primary">
  <p class="p-xsmall">--cx-fg-main</p>
</div>`} />

### Example 2

Lets make `.example-2` a context component using --cx-fg-subtle for its `color` property.

While doing this we're also reserving for `--example-fg-color` if we plan to manipulate it later.

<ScssDocs name="example-2" file="site/src/scss/_context.scss" />

The `%solid-context` selector overrides the `--cx-fg-color` variable with **new** `--cx-fg-main`, we're intercepting it to set `--cx-fg-subtle` as `--cx-fg-color`.

<Example code={`<p class="example-2 p-xsmall">--cx-fg-subtle</p>
<p class="example-2 primary p-xsmall">--cx-fg-subtle</p>
<p class="example-2 primary solid p-xsmall">--cx-fg-subtle</p>`} />

The `%solid-context` selector overrides the component colors.
