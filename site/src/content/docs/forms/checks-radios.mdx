## Design approach

Chassis CSS replaces browser default checkboxes and radios with custom-styled controls that maintain accessibility while providing consistent appearance across browsers and devices.

The `.form-option` component creates controls that look consistent across browsers while supporting accessibility, different sizes, and various layouts.

It is possible to use both the traditional sibling `<input>` and `<label>` elements connected with `id` and `for` attributes and nested inputs within label elements for more compact markup. Both approaches look identical and maintain full accessibility, letting you choose the pattern that works best for your project.

## Checkboxes

Checkboxes allow users to select multiple options from a list. Chassis CSS enhances standard checkboxes with consistent styling and clear state indicators while maintaining accessibility.

<Example code={`<div class="form-option">
    <input class="option-input" type="checkbox" value="" id="checkDefault">
    <label class="option-label" for="checkDefault">
      Default checkbox
    </label>
  </div>
  <div class="form-option">
    <input class="option-input" type="checkbox" value="" id="checkChecked" checked>
    <label class="option-label" for="checkChecked">
      Checked checkbox
    </label>
  </div>`} />

### Indeterminate state

The indeterminate state provides a third visual state for checkboxes, useful for indicating partially selected options or group selections. Since HTML has no attribute for this state, it must be set via JavaScript using the `indeterminate` property.

<Example addStackblitzJs class="cxd-example-indeterminate" code={`<div class="form-option">
    <input class="option-input" type="checkbox" value="" id="checkIndeterminate">
    <label class="option-label" for="checkIndeterminate">
      Indeterminate checkbox
    </label>
  </div>`} />

### Disabled state

Apply the HTML `disabled` attribute to indicate that a checkbox is unavailable for interaction. Chassis CSS automatically styles both the checkbox and its label with visual cues to indicate the disabled state.

<Example addStackblitzJs class="cxd-example-indeterminate" code={`<div class="form-option">
    <input class="option-input" type="checkbox" value="" id="checkIndeterminateDisabled" disabled>
    <label class="option-label" for="checkIndeterminateDisabled">
      Disabled indeterminate checkbox
    </label>
  </div>
  <div class="form-option">
    <input class="option-input" type="checkbox" value="" id="checkDisabled" disabled>
    <label class="option-label" for="checkDisabled">
      Disabled checkbox
    </label>
  </div>
  <div class="form-option">
    <input class="option-input" type="checkbox" value="" id="checkCheckedDisabled" checked disabled>
    <label class="option-label" for="checkCheckedDisabled">
      Disabled checked checkbox
    </label>
  </div>`} />

## Radio buttons

Radio buttons allow users to select a single option from a list of choices. Chassis CSS provides visually consistent radio buttons with clear states while maintaining native accessibility features.

<Example code={`<div class="form-option">
    <input class="option-input" type="radio" name="radioDefault" id="radioDefault1">
    <label class="option-label" for="radioDefault1">
      Default radio
    </label>
  </div>
  <div class="form-option">
    <input class="option-input" type="radio" name="radioDefault" id="radioDefault2" checked>
    <label class="option-label" for="radioDefault2">
      Default checked radio
    </label>
  </div>`} />

### Disabled radio buttons

Like checkboxes, radio buttons can be disabled with the HTML `disabled` attribute. This prevents user interaction and applies a visual state that indicates the control is unavailable.

<Example code={`<div class="form-option">
    <input class="option-input" type="radio" name="radioDisabled" id="radioDisabled" disabled>
    <label class="option-label" for="radioDisabled">
      Disabled radio
    </label>
  </div>
  <div class="form-option">
    <input class="option-input" type="radio" name="radioDisabled" id="radioCheckedDisabled" checked disabled>
    <label class="option-label" for="radioCheckedDisabled">
      Disabled checked radio
    </label>
  </div>`} />

## Toggle switches

Chassis CSS offers toggle switches as an alternative UI for boolean (on/off) selections. Switches are built on checkbox markup but with enhanced styling via the `.form-switch` class.

Adding the `role="switch"` attribute improves accessibility by correctly communicating the control's purpose to assistive technologies that support this role. In older assistive technologies, it will simply be announced as a regular checkbox as a fallback. Switches also support the `disabled` attribute.

<Example code={`<div class="form-option form-switch">
    <input class="option-input" type="checkbox" role="switch" id="switchCheckDefault">
    <label class="option-label" for="switchCheckDefault">Default switch checkbox input</label>
  </div>
  <div class="form-option form-switch">
    <input class="option-input" type="checkbox" role="switch" id="switchCheckChecked" checked>
    <label class="option-label" for="switchCheckChecked">Checked switch checkbox input</label>
  </div>
  <div class="form-option form-switch">
    <input class="option-input" type="checkbox" role="switch" id="switchCheckDisabled" disabled>
    <label class="option-label" for="switchCheckDisabled">Disabled switch checkbox input</label>
  </div>
  <div class="form-option form-switch">
    <input class="option-input" type="checkbox" role="switch" id="switchCheckCheckedDisabled" checked disabled>
    <label class="option-label" for="switchCheckCheckedDisabled">Disabled checked switch checkbox input</label>
  </div>`} />

Chassis CSS provides flexible layout options for checkboxes and radio buttons to accommodate different design patterns.

By default, `.form-option` elements stack vertically with appropriate spacing, creating a clean, readable list of options.

<Example code={`<div class="form-option">

    <label class="option-label" for="defaultCheck1">

  </div>

    <input class="option-input" type="checkbox" value="" id="defaultCheck2" disabled>
    <label class="option-label" for="defaultCheck2">

  </div>`} />

<Example code={`<div class="form-option">

### Inline layout

For horizontal layouts, add the `.form-option-inline` modifier class to display multiple options in a row. This is especially useful for brief options that should appear as a group.

## Without visible labels

For interface designs that require controls without visible text labels, omit the wrapping `.form-option` container. While this creates a visually minimal control, remember that all form elements must still have an accessible name for screen reader users (see [forms overview accessibility]([[docsref:/forms/overview/#accessibility]])).

<Example code={`<div>
    <input class="option-input" type="checkbox" id="checkboxNoLabel" value="" aria-label="Subscribe to newsletter">
  </div>

  <div>
    <input class="option-input" type="radio" name="radioNoLabel" id="radioNoLabel1" value="1" aria-label="Select first option">
    <input class="option-input" type="radio" name="radioNoLabel" id="radioNoLabel2" value="2" aria-label="Select second option">
  </div>`} />

## Input within labels

Chassis CSS fully supports nesting input elements inside label elements as a first-class pattern. This creates an implicit association between the input and its label without requiring `id` and `for` attributes, resulting in cleaner, more concise markup while maintaining full accessibility.

<Example code={`<div>
    <label class="form-option option-label">
      <input class="option-input" type="checkbox">
      Checkbox within label
    </label>
  </div>
  <div>
    <label class="form-option option-label form-option-inline">
      <input class="option-input" type="radio" name="radioLabel">
      Radio within label
    </label>
    <label class="form-option option-label form-option-inline">
      <input class="option-input" type="radio" name="radioLabel">
      Another radio within label
    </label>
  </div>
  <div>
    <label class="form-option form-switch option-label">
      <input class="option-input" type="checkbox" role="switch">
      Switch within label
    </label>
  </div>`} />

Both the nested and sibling markup approaches receive identical styling and functionality in Chassis CSS, allowing you to choose the pattern that best suits your project requirements or team preferences.

## Button-styled toggles

Chassis CSS provides a way to style checkboxes and radio buttons as buttons, combining the semantic meaning of form controls with the visual appearance of buttons. These controls use the `.button-option` class on the input and `.button` styles on the corresponding `<label>` elements.

### Checkbox button toggles

Checkbox button toggles allow users to toggle options on and off with a button-like appearance. They maintain the same states and behaviors as standard checkboxes but with enhanced visual styling.

<Example code={`<input type="checkbox" class="button-option" id="button-option" autocomplete="off">
<label class="button primary" for="button-option">Single toggle</label>

<input type="checkbox" class="button-option" id="button-option-2" checked autocomplete="off">
<label class="button primary" for="button-option-2">Checked</label>

<input type="checkbox" class="button-option" id="button-option-3" autocomplete="off" disabled>
<label class="button primary" for="button-option-3">Disabled</label>`} />
<Example code={`<input type="checkbox" class="button-option" id="button-option-4" autocomplete="off">
<label class="button default" for="button-option-4">Single toggle</label>

<input type="checkbox" class="button-option" id="button-option-5" checked autocomplete="off">
<label class="button default" for="button-option-5">Checked</label>

<input type="checkbox" class="button-option" id="button-option-6" autocomplete="off" disabled>
<label class="button default" for="button-option-6">Disabled</label>`} />

<Callout>
From an accessibility perspective, button-styled checkboxes are announced by screen readers as "checkbox checked"/"checkbox not checked" since they maintain their semantic role. This differs from toggle buttons created with the [button component]([[docsref:/components/buttons#button-plugin]]), which are announced as "button"/"button pressed". Choose the appropriate approach based on whether the control's purpose is better communicated as a state change (checkbox) or an action trigger (button).
</Callout>

### Radio button toggles

Radio button toggles provide a button-styled interface for mutually exclusive selections. They offer the same semantic meaning as standard radio buttons but with an enhanced visual appearance that may be more suitable for certain UI patterns.

<Example code={`<input type="radio" class="button-option" name="options" id="option1" autocomplete="off" checked>
<label class="button secondary" for="option1">Checked</label>

<input type="radio" class="button-option" name="options" id="option2" autocomplete="off">
<label class="button secondary" for="option2">Radio</label>

<input type="radio" class="button-option" name="options" id="option3" autocomplete="off" disabled>
<label class="button secondary" for="option3">Disabled</label>

<input type="radio" class="button-option" name="options" id="option4" autocomplete="off">
<label class="button secondary" for="option4">Radio</label>`} />

<Example code={`<input type="radio" class="button-option" name="options-base" id="option5" autocomplete="off" checked>
<label class="button default" for="option5">Checked</label>

<input type="radio" class="button-option" name="options-base" id="option6" autocomplete="off">
<label class="button default" for="option6">Radio</label>

<input type="radio" class="button-option" name="options-base" id="option7" autocomplete="off" disabled>
<label class="button default" for="option7">Disabled</label>

<input type="radio" class="button-option" name="options-base" id="option8" autocomplete="off">
<label class="button default" for="option8">Radio</label>`} />

### Style variants

Chassis CSS's button-styled toggles support all visual variants available to standard buttons. This includes outlined styles, which provide a lower-emphasis alternative that still clearly communicates the interactive nature of the control.

<Example class="button-examples" code={`<input type="checkbox" class="button-option" id="button-option-outlined" autocomplete="off">
<label class="button primary" for="button-option-outlined">Single toggle</label>

<input type="checkbox" class="button-option" id="button-option-2-outlined" checked autocomplete="off">
<label class="button outline secondary" for="button-option-2-outlined">Checked</label>

<br>

<input type="radio" class="button-option" name="options-outlined" id="success-outlined" autocomplete="off" checked>
<label class="button outline success" for="success-outlined">Checked success radio</label>

<label class="button outline danger" for="danger-outlined">Danger radio</label>`} />

## Accessibility considerations

Chassis CSS's checkbox and radio components are designed with accessibility in mind, but there are several best practices to follow for optimal user experience:

### Grouping related controls

When presenting multiple related options, use the `<fieldset>` and `<legend>` elements to create a semantic grouping:

```html
<fieldset>
  <legend>Choose your delivery preferences:</legend>
  <div class="form-option">
    <input class="option-input" type="checkbox" id="optEmail">
    <label class="option-label" for="optEmail">Email notifications</label>
  </div>
  <div class="form-option">
    <input class="option-input" type="checkbox" id="optSMS">
    <label class="option-label" for="optSMS">SMS notifications</label>
  </div>
</fieldset>
```

### Focus visibility

Chassis CSS provides visible focus indicators for all form controls. Do not remove these indicators, as they are essential for keyboard users to navigate your forms.

### State communication

The different visual states of checkboxes and radio buttons (checked, unchecked, indeterminate, disabled) are designed to be perceivable through:

- Visual design (color, shape changes)
- High contrast ratios between states
- Non-reliance on color alone to communicate state

For more detailed accessibility guidance, see the [forms overview accessibility]([[docsref:/forms/overview/#accessibility]]) section.

### Theming examples

You can override these variables in your own CSS to create custom-themed controls. For example:

```css
:root {
  --chassis-option-checked-bg: #9c27b0;
  --chassis-option-checked-border-color: #7b1fa2;
  --chassis-option-focus-box-shadow: 0 0 0 0.25rem rgba(156, 39, 176, 0.25);
}
```

This would create purple-themed checkboxes and radio buttons while maintaining all the functionality and accessibility features of the standard controls.
